vue电商后台管理登录退出功能

vue.js项目
vue.js电商后台管理系统
发布日期:   2020-03-09

项目初始化

使用vue-cli脚手架进行项目的初始化,添加element-ui插件和vue-router插件，添加axios第三方开发依赖，使用git将项目部署到远程代码管理仓库协同开发github或者码云，运行后台服务器，将数据库导入，使用postman进行接口测试，项目前期准备工作完成

使用git多人协同开发将项目代码托管到github远程仓库，vue脚手架创建项目内置webpack来开发vue项目

登录功能

在不存在跨域问题的时候，http的无状态性可以通过session和cookie来维持用户的登录状态，在创建vue项目的时候是运行在另外一个端口号，因此涉及到跨域问题，使用token来维持的用户的登陆状态

  1. 在第一次输入用户名和密码，将登录请求发给服务器端验证是否有该用户如果有就将用户唯一的token命令和登录信息响应给客户端表示登陆成功，客户端存取该用户的token在之后每次请求时携带该token，服务器端验证是否有这个token,如果有就保持联系展示用户登录之后才能看到的数据

  2. 使用element-ui进行登陆页面的布局,创建登录分支,每次开发一个新功能后都需要创建分支在当前分支中开发,最后合并到主分支中,先查看暂存区是否干净git status，然后使用git checkout -d 分支名 创建并进入分支,git branch查看所有的分支

  3. 梳理项目结构,删除使用脚手架创建项目默认的一些组件和路由规则

  4. 创建登录组件login.vue，将组件导入路由文件中,设置/login路由规则,在app跟组件中添加路由占位符,当访问/login时会讲登录组件渲染到app跟组件中,将/网站根目录重定向到/login

  5. 设置登录组件的背景颜色,画登录盒子样式,居中显示,绘制头像区域样式

  6. 使用element-ui的表单组件，在登陆表单中需要用到的element-ui组件有Form组件，表单域Form-item组件，Input组件，和Button组件，将这些组件按需导入,修改样式

  7. 给表单框添加字体图标

  8. 表单的数据绑定，给form组件绑定一个model属性,该属性为表单对象在data中定义该表单对象，该表单里的每一个表单域的值应该双向绑定到该表单对象的具体属性下

  9. element-ui提供了表单组件的验证功能,在form组件中绑定rules属性,该属性为一个验证规则对象,属性名为定义的规则名,属性值为数组,可以定义多条规则,然后需要使用的form-item组件标签使用prop属性对应验证规则名

    rules: {
          name: [
              //required为必填，message错误信息，trigger为什么时候触发验证
            { required: true, message: '请输入活动名称', trigger: 'blur' },
            { min: 3, max: 5, message: '长度在 3 到 5 个字符', trigger: 'blur' }
          ],

  10. 点击重置按钮，重置表单值,使用表单组件引用对象下的resetFields方法可以将所有的字段重置为初始值,和移除验证结果,获取form组件的引用对象可以给form组件设置一个ref属性,该属性关联一个表单组件引用对象,使用this.$refs能获取到该表单组件的引用对象

  11. 点击登录按钮，使用ref关联表单组件的引用对象,调用引用对象下的validate方法进行在登陆之前的预验证，该方法接受一个回调函数的参数,回调函数第一个参数是一个布尔值为验证的结果,如果为true说明验证成功允许发送登录请求

  12. 如果预验证成功说明可以发送axios登录请求,导入axios对象，然后将axios方法挂载到Vue构造函数的原型对象中，这样所有的组件中都可以使用this来访问到axios方法了,配置axios的请求根路径，如果请求的返回值的status不等于说明登录失败,等于200才是登陆成功

  13. 在验证完成后使用Message全局弹框组件设置登陆是否成功的弹框消息,该组件需要全局使用需要挂载在Vue的原型对象中使用其中的一些弹框方法

  14. 在登录之后将服务器响应的信息中token数据存储在浏览器的sessionStorage中,sessionStorage在浏览器关闭就会使释放空间,我们只需要保证浏览器开着的时候服务器端与客户端能维持登录状态即可，每次发送请求都会客户端携带sessionStorage中的token信息进行访问,服务器端验证成功后就可以访问登陆权限之后的页面了，跳转到home后台管理界面路由

  15. 设置路由访问权限,设置路由守卫只有登录了之后才能访问/login之外的页面,没有登陆就强制跳转到登陆页面

    // 挂载路由导航守卫,控制页面访问权限
router.beforeEach((to, from, next) => {
  // to 要访问的地址
  // from 从哪里跳转的地址
  // next() 直接放行 next('/login)强制跳回login路由
  if (to.path === '/login') return next()
  // 获取客户端token判断是否登录
  const tokenStr = window.sessionStorage.getItem('token')
  // 如果没有token强制跳回login页
  if (!tokenStr) return next('/login')
  next()
})

退出功能

清除本地浏览器sessionsStorage的token缓存即可,后续请求不会携带该数据,在跳转到/login页面

Form组件的使用

element-ui的Form组件的基本用法,在vue需要用到表单组件的就可以使用Form组件，每一个表单域被Form-item组件包裹，每一个Form-item组件下就是一个表单控件组件，例如Input组件 Button组件 select组件

添加字体图片如果时input组件可以通过添加属性在prefix-icon前面添加字体图标或者后面suffix-icon等于对应的字体图标值,如果element-ui中没有的字体图标可以去第三方字体图标库中下载

Form组件的数据绑定,给form标签绑定:model属性该属性为表单绑定对象,该表单组件中所有的表单控件都需要双向绑定v-model到表单绑定对象中

Form组件的表单验证,给form标签绑定:rules属性该属性为表单验证规则对象，属性值是对应的规则数组，一个相应的规则名中可以有多条规则，在使用规则的时候给需要的form-item组件标签添加prop属性为规则名即可使用form表单组件的规则验证

Form组件的表单重置，可以使用表单组件的对象下的resetFields来初始化表单控件的值和清除验证规则，给form表单添加ref属性,该属性为表单的引用对象，可以在vue组件实例的$refs下来获取该表单引用对象使用resetFields来进行重置表单

Form组件的预验证，使用表单引用对象下的validate方法，该方法需要传入一个回调函数,在回调函数中有两个参数第一个是布尔值为是否验证成功，第二个是对象为未验证通过的字段，一般只需要第一个参数，如果第一个参数为true说明验证成功那么就是使用axios发送登录请求了

ref属性为表单添加表单引用对象用来调用表单组件下的一些方法
