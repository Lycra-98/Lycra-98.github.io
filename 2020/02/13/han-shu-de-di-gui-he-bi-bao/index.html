函数的递归

函数的递归是指在函数中调用函数自己

从定义上我们就可以看出,一个函数在外部被调用，在里面还需要调用自己，那么它就会在内部一直反复调用自身，因此我们要限制递归的条件，不能无限被调用。

递归的条件

递归有两大条件

  1. 递归的停止条件,在什么时候停止调用自己,但是只有停止条件不行,我们还需要每一次递归后都一步步接近这个条件，所以一般递归问题我们都会反向思考,每次递归的过程都会一步一步的将问题所简化
  2. 调用递归的条件,就是我们想要在什么时候调用自己

递归的举例

我对递归的理解就是,在你执行一个函数的过程中,中间需要你去调用另外一个函数,但是本身并没有被执行完，那么就会暂时将自己压下来去等待中间函数的调用,在中间函数执行过程中,又没有执行完成就去调用另一个函数,当前函数又会被压下,如此反复循环,等到达终止条件时,最后一个调用执行完成会从当前为起点往回依次执行完之前之前被压下的函数

function f(n) {
            console.log(n);
            n = n - 2;
            if(n > 0) {
                f(n);
            }
            console.log(n);
        }
        f(10);

斐波那契数列
1,1,2,3,5,8,13
​ 第一项1 第二项1 第三项2 第四项3 第五项5 第六项8 第七项13
​ 从第三项开始2 = 1 + 1 第二项 + 第一项
​ 第四项3 = 2 + 1 第三项 + 第二项
​ 因此从第三项开始后面的数等于前面两项之和
​ 递归公式
​ f(3) = f(2) + f(1) f(4) = f(3) + f(2)
​ ….
​ f(n) = f(n -1) + f(n - 2)
​ 递归结束条件为n = 1 和 n = 2


        function fibonacc(n) {
            if (n == 1 || n == 2) {
                //第一项和第二项直接返回1
                return 1;
            } else if (n > 2) {
                //当n从第三项开始,后面一项等于前面两项之和
                return fibonacc(n - 1) + fibonacc(n - 2);
            }
        }
        console.log(fibonacc(6));
        console.log(fibonacc(10));

深拷贝递归

        //深拷贝是多层拷贝,如果拷贝的是引用数据类型,会重开一个内存空间把数据完全拷贝到新对象中,而不是只拷贝内存地址
        //因此内存地址不是指向同一个，修改其中一个值不会影响另一个
        let cq = {
            uname: 'cq',
            age: 18,
            msg: {
                sex: 'man',
                birth: '10/24'
            },
            color:['blue', 'pink']
        }

        let o = {}

        //因为深拷贝拷贝多层,见到数据类型直接拷贝值,复杂数据类型需要遍历重新赋值给新对象
        //外层对象也需要遍历,因此内层如果是复杂数据类型需要重复遍历,就需要使用递归
        //封装深拷贝函数两个参数,第一个参数拷贝给谁(目标对象),第二个参数原始对象即被拷贝的对象
        function deepCopy(newObj, oldObj){
            //遍历被拷贝的对象
            for(let k in oldObj) {
                //获取对象的属性值
                let item = oldObj[k];
                //因为对象中存了不同数据类型的数据，字符串、数字、数组、对象等
                //如果是简单数据类型直接复制,将值直接赋值给目标对象即可newObj[k] = item
                //如果是复杂数据类型，要进入其中继续遍历,将里面的值取出来一个个赋值
                //判断item的类型
                if (item instanceof Array) {
                    //如果是数组
                    newObj[k] = [];
                    deepCopy(newObj[k], item);
                } else if (item instanceof Object) {
                    //如果是对象
                    newObj[k] = {};
                    deepCopy(newObj[k], item);
                } else {
                    //就是简单数据类型
                    newObj[k] = item;
                }
            }

            return newObj;
        }

        console.log(cq);
        console.log(deepCopy(o, cq));
        //深拷贝修改其中一个引用数据类型的值不会改变另一个
        o.color[0] = 'red';
        console.log(o);//改变了
        console.log(cq);//原来的没有改变

猴子吃桃问题

​ 猴子吃桃问题：猴子第一天摘下若干个桃子，当即吃了一半，还不过瘾，又多吃了一个，
​ 第二天早上又将剩下的桃子吃掉一半，又多吃了一个。以后每天早上都吃前一天剩下的一半零一个。
​ 到第10天早上想再吃时，见只剩下一个桃子了。求第一天共摘多少个桃子？
​ 第一天的桃子数量,等于第二天的桃子数量+1乘以二 f(1) = (f(2) + 1) * 2
​ 第二天的桃子数量,等于第三天的桃子数量+1乘以二 f(2) = (f(3) + 1) * 2
​ 第三天的桃子数量,等于第四天的桃子数量+1乘以二 f(3) = (f(4) + 1) * 2
​ …第十天的桃子数量,等于1 f(10) = 1
​ 每天的桃子数量等于后一天桃子数量+1 乘以2 从第十天可以倒推第一天的 当n==10停止递归


        function f(n) {
            //第十天停止递归
            if (n == 10) {
                return 1;
            }
            return (f(n + 1) + 1) * 2;
        }

        console.log(`第一天一共摘了${f(1)}个桃子`);

闭包

闭包的简单理解就是函数的嵌套,一个函数能访问到另一个函数中的局部变量。

这里有一个局部变量的概念，就是局部作用域即函数作用域,一般来说函数作用域在函数调用的时候被创建,这个时候这个作用域的变量会被开辟一个内存空间去保存,在函数调用完成后，函数作用域就会销毁,同时该变量也会被销毁，释放这个内存空间，这是一般情况。

闭包就是延续了变量的生命周期，使其嵌套的函数能够访问到这个局部变量，尽管这个函数已经被调用,在函数调用完成时,闭包现象就是能继续访问里面的局部变量,说明变量没有被销毁，还存在意味着内存空间还存着它，造成了内存泄露。
