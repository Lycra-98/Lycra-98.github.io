Express框架初步使用

-Express -node.js
Express框架
发布日期:   2020-02-18

Express框架

express框架提供了一套完整用于开发web网站的api，让开发者创建web服务器比原生node.js更加容易。作为第三方模块,需要用npm下载并在项目中引用,会返回一个网站服务器对象就不用再添加http系统模块来创建了,使用框架后如果没有访问了没有设置的路由,不会跳到没有此网站,而且会响应connot get 该路由地址

初步使用

//引入express框架
const express = require('express');
//创建网站服务器
const app = express();

app.get('/', (req, res) => {
    //响应用send方法
    //1.send方法内部会检测响应内容的类型
    //2.send方法会自动设置http状态码
    //3.send方法会帮我们自动设置响应的内容类型和编码
    res.send('hello world');
})
app.get('/list', (req, res) => {
    //可以直接响应json对象
    res.send({name: '曹强', age: 20});
})

app.listen(3000);
console.log('服务器创建成功');

中间件

​ 中间件是用来处理浏览器的请求的一些方法(就是拦截请求做一些处理再给出响应),当浏览器发送请求时候,由中间件来处理该请求，该请求可以同时被多个中间件请求,中间件分为express提供的例如get方法和post方法创建路由地址,也有开发人员写的,例如创建路由的处理函数就是由开发人员去写的,同一个请求是可以由多个中间件去处理的,不过默认从上向下一次匹配中间件,匹配成功了就会终止匹配,因此需要处理同一个请求设置不同的中间件时，需要next方法将请求传到下一个中间去处理

//处理同一个get请求
app.get('/request', (req,res,next) => {
    req.name = 'zhangsan';
    next();
})
app.get('/request', (req, res) => {
    res.send(req.name);
})

app.use方法接收浏览器的所有请求不管是get还是post,因此请求匹配成功一个就不会走下一个中间件,因此该中间件写在最前面来接收所有请求,如果该中间件没有做出相应,也没有用next()将请求控制权给下一个中间件,那么浏览器就会卡住等待服务器响应

//接受了浏览器的所有请求,不管是get还是post
app.use((req, res, next) => {
    //接受了所有请求,如果没有做出响应,也没有next交给下一个请求,就会卡在这里
    console.log('我是走app.use的请求');
    next();
})
//当请求地址为/request的时候先走上一个请求再走这一个请求
app.use('/request',(req, res, next) => {
    console.log('我是走app.use路径为/request的请求');
    next();
})

中间件的一些应用场景(简单应用)

  1. 用户登录,在用户登陆成功之前，判断用户是否登陆成功,没有登陆成功拦截请求,成功了就next()
  2. 网站维护时,接受任何请求地址直接给出响应
  3. 404访问的资源不存在,在最后定义，前面设置的路由没有访问到,就说明没有该资源,就直接响应404
//网站维护公告,拦截所有请求给出响应
// app.use((req,res,next) => {
//     res.send('网站正在维护中...');
// })

app.use('/admin', (req,res,next) => {
    //表示用户登陆是否成功
    let login = true;
    if (login) {//登陆成功走下一个中间件
        next();
    } else {//登陆失败直接给出响应
        res.send('请登录才能访问该页面');
    }
})

//说明登陆成功
app.get('/admin', (req,res) => {
    res.send('登陆成功,您可以访问该页面');
})

//404页面
app.use((req,res,next) => {
    //设置404状态码
    res.status(404).send('您访问的页面不存在');
})

错误处理中间件

一般在客户端发送请求时,比如读取文件或者读取数据库失败，都会有错误信息,我们需要读取错误信息，并且设置对应的解决办法，express给我们提供了错误处理中间件,分为同步处理和异步处理,同步处理,在发生错误时,直接抛出错误即throw new Error然后调用错误处理中间件读取错误设置响应,异步处理即载文件读取失败或者读取数据库失败,用next()传递错误对象参数,触发错误处理中间件读取错误对象的信息,调用了错误处理中间件并设置响应后,文件就可以在报错后正常运行

app.get('/index', (req, res, next) => {
    //同步抛出错误,触发错误处理中间件
    // throw new Error('程序发生未知错误');
    //处理异步api错误需要用next()传递错误对象,来调用错误处理中间件
    fs.readFile('./01.js', 'utf8', (err, result) => {
        if (err != null) {//说明有错误用next传递错误对象调用错误处理中间件
            next(err);
        } else {
            res.send(result);
        }
    })

})

//错误处理中间件,默认捕捉同步信息
app.use((err, req, res, next) => {
    //错误对象,读取错误信息就是错误对象下的message属性
    res.status(500).send(err.message)
})

异步api的错误信息都是通过回调函数去处理,支持promise对象的用catch可以捕获错误信息

try catch可以捕获异步函数以及其他同步代码在运行时的错误

try{可能放生错误的异步函数}catch(错误对象){next(err) 触发错误处理中间}

一般支持promise对象的都能写成异步函数形式,不支持的就需要用util这个模块中的promisify()这个方法去包装才能写成异步函数形式

//引入express框架
const express = require('express');
const promosify = require('util').promisify;
const fs = require('fs');
//包装文件读取api
const readFile = promosify(fs.readFile);
//创建网站服务器
const app = express();

app.get('/index', async (req, res, next) => {
    try{
        await readFile('./01.txt');
    } catch(err) {
        next(err);
    }
})

构建模块化路由

将请求不同主页面的路径交给不同的路由对象去处理，这样不同的路由对象就可以单独放在js文件中作为单独的模块导出即可,可以在主入口文件中导入,并为不同的路由对象匹配路径

处理get和post请求参数

get请求参数

app.get('/index', (req,res) => {
    //get请求参数,直接通过请求对象req.query属性获得,自动转化成对象形式
    res.send(req.query);
})

post请求参数

//引入处理post请求参数的第三方模块
const bodyparser = require('body-parser');
const app = express();

//拦截所有请求,如果有post请求带来的请求参数,就用urlencoded将其转化为对象,添加给请求对象的body属性,如果没有post请求参数会自动next()交给下一个中间件
//extended:false 使用querystring模块处理
//extended:true 使用qs第三方模块处理请求参数
app.use(bodyparser.urlencoded({extended: false}));//调用urlencoded方法返回值也是一个函数,带参数req,res,next的

app.post('/add', (req, res) => {
    res.send(req.body);
})

app.use的使用,在app.use中传递函数的调用

//可以在app.use方法里面调用定义好的函数,可以带上实参来控制返回的函数
//但是这个调用的函数要返回一个带有参数req,res,next的函数
app.use(fn({a: 1}));

function fn(obj) {
    return function (req,res,next) {
        if (obj.a == 1) {
            console.log(req.url);
        } else {
            console.log(req.method);
        }
        next();
    }
}

路由参数

app.get('/index/:id/:name/:age', (req,res) => {
    //路由参数的表现形式为/123而不是分隔,这种方法也可以当作get请求参数
    //使用req.params属性读取路由参数,内部自动转化为对象形式
    res.send(req.params);
    //需要使用/index/id/name/age来访问该路由
})

静态资源访问


//拦截所有请求通过express内置的static(静态资源路径)访问静态资源,如果不需要访问静态资源会自动next下一个中间件
app.use('/static',express.static(path.join(__dirname, 'public')));

使用模板引擎

下载art-template和express-art-template这两个模板引擎

//告诉express框架使用什么模板引擎来渲染什么后缀的模板文件
//1.模板后缀2.模板引擎
app.engine('art',require('express-art-template'));
//告诉express框架模板存放的位置,配置模板位置,第一个值是规定的配置模板路径需要写的,第二个值是模板所在路径
app.set('views',path.join(__dirname,'views'));
//告诉express框架模板默认的后缀是什么,第一个参数是规定这样写的
app.set('view engine', 'art');

app.get('/index', (req,res) => {
    //使用res.render()方法使用模板引擎
    //将路径和后缀拼接好
    //直接将拼接好的结果相应给了客户端,不需要设置变量接收
    res.render('index', {
        msg: 'index'
    })
})
//将公共数据添加到locals对象下,每个模板都可以访问到该数据locals.变量名 = 数据
app.locals.users = [{
    name: 'zhangsan',
    age: 20
},{
    name: 'lisi',
    age: 18
}]

总结

  1. express框架提供了一套完整构建web网站结局方案,比起使用原生更加快捷方便，结构看起来更加清晰
  2. 创建服务器不需要引入http模块，直接调用require框架返回的方法创建服务器对象，监听路有不需要判断路径和请求方式什么的，可以直接使用服务器对象下的get、post、use方法，使用res.send()响应给浏览器不需要往响应头再去指定类型和编码了
  3. 模块化路由，可以根据请求路径不同在不同模块中(不同js文件)创建路由对象具体的请求响应步骤在该文件中执行,只需要导入模块对象,在主入口文件中导入指定对应路由的请求地址即可
  4. 处理get请求参数,直接使用请求对象中的.query属性即可,post请求参数导入请三方模块body-parser返回一个处理参数的对象,在use方法中调用该对象下的urlcoded({extended:false})即可在请求对象中使用body属性获取请求参数对象
  5. 路由参数，就是指定路由地址/参数1/参数2，而不是以？分隔的参数，通过请求对象的params属性获取
  6. 静态资源访问直接在use方法中调用框架提供的static()方法即可，参数填静态资源的路径s
  7. 使用模板引擎,下载express-art-template 和 atr-tempalate，做四件事
    • 设置用什么模板引擎来渲染什么后缀的模板文件engine方法
    • 用set方法配置模板文件的根目录
    • 用set方法配置模板文件的默认后缀名
    • 用响应对象中的render(‘文件名不需要后缀’, ‘数据’)方法即可渲染想要的模板文件到页面中
    • 公共数据需要所有模板都能访问到的数据,可以属性的形式添加到locals对象下
  8. 中间件的使用,即框架提供的一些方法用来处理客户端请求的,用next()方法可以将请求处理权交给下一个中间件,协同处理同一请求,错误处理中间件,用来在发生错误时调用,使文件再发生错误时能正常运行,同步通过throw new error正常抛出错误调用,异步通过next()传入错误对象调用
