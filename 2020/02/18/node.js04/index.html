node模板引擎

node.js
发布日期:   2020-02-18

模板引擎

使用模板引擎是为了方便拼接html文件字符串,目前使用最多的是art-template模板引擎

基本使用,先下载第三方模块art-template,然后引用会返回一个方法,这个方法用来拼接模板中的字符串,模板就是需要拼接的html文件,只不过一般改成art后缀

//返回一个方法
const template = require('art-template');
const path = require('path');

const views = path.join(__dirname, 'views', 'index.art');
//方法第一个参数为模板的路径绝对路径,第二个参数为要在模板中显示的数据,对象类
//返回是拼接好的字符串
const html = template(views, {
    name: '张三',
    age: 20
})

模板语法

标准语法

原始语法 <%= 数据 %>

模板引擎默认不识别数据中的html标签，如果需要识别html标签需要原文输出

标准语法

原始语法 <%- 数据 %>

条件判断

标准语法
{{if 条件}} ... {{/if}}
{{if 条件1}} ... {{else if 条件2}} ... {{/if}}

原始语法
<% if (条件) { %> ... <% } %>
<% if (条件) { %> ... <% } else if (条件2){ %> ... <% } %>

循环

标准语法$index表示索引,$value表示数组元素
{{each 数据}}
    {{$index}} {{$value}}
{{/each}}
原始语法
<% for(var i = 0; i < 数据.length; i++) { %>
    <%= i %> <%= 数据[i]>
<% } %>

子模版

子模版将网站公共的区块(头部、尾部)抽离到单独的文件中,然后再想要使用子模版的页面引入子模版即可

标准语法
{{include '模板路径'}}
原始语法
<% include('模板路径')%>

模板继承

  1. 把html骨架模板单独拿出来放在一个模板里,因为所有页面都需要他,然后每个页面在使用的时候继承该html骨架模板即可

  2. 由于每个页面模板需要填写的link以及内容不同,所以在骨架模板中可以用block ‘填充部分名字’ /block表示该位置需要你继承时填入你自己所需要的,例如这是一份骨架模板,页面标题和link引入文件部分和内容部分,需要你继承的时候自己去填写

  3. <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    {{block 'title'}}{{/block}}<-- 需要填充标题 -->
    {{block 'link'}}{{/block}}<-- 需要填充引入的文件 -->
</head>
<body>
    {{block 'content'}}{{/block}}<-- 需要填充内容 -->
</body>
</html>

    继承骨架模板,继承时用 extend ‘骨架模板路径’,同时用 block ‘填充部分名字’ /block,把你这个页面需要的内容填充完整

    {{extend './common/layout.art'}}
{{block 'title'}}<title>首页</title>{{/block}}<-- 填充了标题 -->
{{block 'link'}}
<link rel="stylesheet" href="style.css"><-- 填充了引入的css文件 -->
{{/block}}

{{block 'content'}}<p>{{msg}}</p>{{/block}}<-- 填充了内容 -->

模板配置

1.模板中不能直接使用第三方模块的方法,需要像模板中导入变量

template.defaults.imports.变量名 = 变量值

const format = require('date-format');//引入处理时间的第三方模块

//将format方法导入模板中,那么模板中就可以使用该方法
template.defaults.imports.format = format;

2.配置模板的根目录,如果模板都在同一目录下,我们可以设置根目录,那么每次使用template是的路径就可以直接写模板文件名,还可以配置模板的默认后缀

//配置模板根目录,模板根目录为views文件夹
template.defaults.root = path.join(__dirname, 'views');
//配置模板的默认后缀,将模板默认后缀设置为.art
template.defaults.extname = '.art'

//写路径的时候直接写views下文模板名即可,默认后缀为.art
const html = template('06', {
    date: new Date()
})

总结

模板引擎,在我们想要网站动态将html文件响应给页面时,之前的做法是将整个HTML文件拼接字符串当作变量响应给客户端,现在可以直接用模板引擎在html页面进行数据的操作,然后返回直接拼好的字符串. 在模板中我们可以原始方法和标准方法对数据进行处理,也可以使用提供的一些方法在模板说写js语法,将html页面里的数据变成动态的
